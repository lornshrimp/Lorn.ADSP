1. 页面结构设计 (信息架构 & 布局)

1.1 仪表盘布局与数据可视化

  • 三层信息层级：Top KPI (即时认知) → 分析图表 (趋势/组成) → 行为上下文 (告警 & 活动流)
  • 可配置网格：支持 12 列自适应重排，组件“重量”分级 (KPI=1, Chart=2, Table=3)
  • 实时模块采用渐进刷新 + Diff 标记 (绿色上涨 / 红色下跌)，峰值时降级刷新频率
  • 关联导航：告警项可一键跳转对应计划详情 (Deep Link)

1.2 账户管理页信息层级

  1. L1 摘要：财务与运行状态 (预算 / 消耗 / 状态 / 标签)
  2. L2 结构与权限：组织 / 用户 / 角色 / 标签实体关系
  3. L3 运营维度：计划 / 历史变更 / 日志 / 资质 Tab 切换

列表 + 详情双栏结构：左侧持久导航 (账户树/搜索/过滤)，右侧主区自适应滚动，Tab 内延迟加载。

1.3 投放管理操作流程

  • 向导式 7 步：支持中途保存草稿 / 步骤校验 / 冲突检测 (预算冲突、定向重叠)
  • 批量模式：表格 + 侧栏编辑器，变更模拟 (显示影响条目数 / 冲突预警)
  • 即时验证：每步提交触发规则引擎返回校验提示 (Inline + 顶部摘要)

1.4 报表页面图表布局

  • 主轴：全局对比区 (关键指标多轴趋势) + 多图栅格 (支持拖拽排序 / 模板保存)
  • 侧栏：维度/指标/过滤工具，允许折叠；筛选变化 → 显示“待应用”状态，批量应用
  • 图表类型规范：曲线趋势 / 柱状对比 / 热力 (地域) / 环图占比 / 箱型 (创意表现) / Sparkline 网格 (小时粒度)

2. 交互体验设计

2.1 多账户切换便捷操作

  • 主导航内联选择器 + 快速过滤弹出层 + 近期访问缓存 (本地优先 + 服务端 Freq 排序)
  • 键盘支持：Ctrl + K 呼出全局命令面板，输入账户名模糊匹配 → Enter 切换
  • 切换优化：预拉取新账户关键权限 + KPI Skeleton Loading ≤ 400ms 反馈

2.2 投放计划批量管理

  • 批量模式激活条件：≥1 勾选 → 顶部浮动批量操作条 (持久可见)
  • 冲突策略：批量修改时显示“冲突/不适用”字段 (灰显 + Tooltip 说明)
  • 撤销：操作成功后 8 秒内支持 Undo (队列 + 状态回滚)

2.3 实时数据刷新策略

类型 策略 降级规则 备注
KPI 实时 WebSocket 差量 连接断开→30s 轮询 Diff 高亮 1.2s
趋势图 30s 合并刷新 浏览器隐藏→暂停 恢复后补帧
长列表 120s 轮询 CPU 忙→延迟 60s 手动刷新优先
报表导出 手动触发 - 后台任务进度

2.4 异常告警提醒机制

  • 分级展示：Critical=红 Badge + Toast；Warning=侧边抽屉；Info=消息中心
  • 聚合：相同维度 & 规则 ID & 5 分钟内合并计数 ("预算超支 x3")
  • 行动引导：告警卡片包含“跳转计划 / 静音 / 创建自动化”次级操作

3. 视觉风格指南

3.1 色彩搭配方案 (B2B 企业级)

用途 Token 颜色 示例
主品牌 --color-brand-primary #1F65D6 Primary
强调/渐变 --color-brand-secondary #2674F0 Secondary
成功 --color-accent-success #1FAD66 Success
警告 --color-accent-warning #F5A623 Warning
危险 --color-accent-danger #E54848 Danger
信息 --color-accent-info #2CB4C9 Info
背景层次 Neutral 50-900 #F8FAFC~#111826 BG

配色策略：主色 ≤ 15% 面积，用于主操作与聚焦；中性色承担信息密度；状态色仅用于语义真实状态，不做品牌装饰。

3.2 数据图表统一规范

  • 调色板顺序：Primary → Success → Info → Warning → Danger → Neutral
  • 统一留白：左右 16px，图表网格线不超过 30% 不透明度
  • Sparkline 高度固定 40px；多轴曲线主轴加粗 2px
  • 单位显示：数值旁统一显示缩写 (K/M)；Tooltip 展示完整值 + 环比/同比

3.3 图标风格与状态标识

  • 线性 2px 描边 + 圆角；填充仅用于高强调行动点
  • 状态 Token：圆角 12px Pill，对比度满足 WCAG AA (文本≥4.5)
  • Loading：骨架屏 + 条形动画 (线性渐变 1400ms 循环)

3.4 空白状态引导设计

  • 结构：图形(可选) / 主文案 (18px/600) / 辅助描述 / 主行动按钮 / 次级替代路径
  • 策略：强调“下一步”操作 (创建 / 导入 / 清除筛选) 而非解释性文字堆叠
  • 反馈闭环：首次创建成功 → 演示引导 (Coach Marks 3 步)

4. 关键交互模式微规范

  • 快捷键：? 打开快捷键帮助；G + D (Dashboard)、G + C (Campaigns)
  • 可编辑表格：单元格获得焦点时 Enter=编辑, Esc=取消, Tab=提交并到下一个
  • 搜索建议：≥2 字符触发；首字母拼音匹配；最多 8 条 + “更多结果...” 链接
  • 错误提示：字段下方 12px 文本 + 图标；全局错误聚合在顶部 Summary Bar

5. 可访问性 (A11y) & 性能

  • 焦点环：统一 2px 品牌色外发光 (outline-offset:2px)
  • 对比度：文本与背景对比度≥4.5；装饰性图标 aria-hidden
  • 懒加载：首屏关键模块优先；报表区按需渲染 (IntersectionObserver)
  • 骨架屏：预计渲染 >300ms 的图表组件使用骨架骨条
